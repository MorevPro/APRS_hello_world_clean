# Задача 3: Читаемый файл настроек APRS (TXT)

**Родительская задача**: [task-aprs-spec-compliance.md](task-aprs-spec-compliance.md)  
**Статус**: Новая  
**Приоритет**: Высокий  
**Зависимости**: [task-aprs-02-keyboard-save.md](task-aprs-02-keyboard-save.md) (общая модель полей)

---

## Цель

Хранить пользовательскую конфигурацию APRS в **отдельном читаемом текстовом файле** `.txt`, чтобы настройки можно было править вручную на ПК, бэкапить и версионировать, а приложение при запуске читало те же параметры, что и после ввода с клавиатуры.

## Контекст проекта

- Репозиторий: корень `APRS_hello_world_clean`, сборка через **ufbt/fbt**; на устройстве данные обычно лежат на **внешней флешке** (`/ext/...`).
- Жёстко заданные сейчас поля см. в `aprs_hello_world_clean_app.c` (`aprs_address`) — файл конфигурации должен **заменить** эту роль как источник истины после первой записи.
- Дополнительно в репозитории имеет смысл положить **пример/шаблон** конфигурации (читаемый `.txt`), чтобы было ясно, какие ключи поддерживаются.

## Требования

### Расположение

- [ ] Зафиксировать **один путь** на SD-карте Flipper, например:  
  `/ext/apps_data/aprs_hello_world_clean/aprs_config.txt`  
  (имя каталога согласовать с именем приложения в `application.fam` / ID FAP).
- [ ] При первом запуске создать каталог при необходимости и **записать файл по умолчанию** (или создать при первом сохранении из UI).

### Формат файла

- [ ] **Человекочитаемый** формат: рекомендуется `ключ=значение` по строке, комментарии `#` в начале строки — по желанию.
- [ ] Поля должны покрывать те же данные, что и UI: `source_call`, `source_ssid`, `destination_call`, `destination_ssid`, path (`path1_call`, `path1_ssid`, `use_path1`, при необходимости path2), координаты, bearing, speed, текст статуса.
- [ ] Кодировка: **ASCII/UTF-8 без BOM**; только символы, допустимые для APRS в соответствующих полях.
- [ ] Явно документировать в шаблоне в репозитории единицы измерения (градусы/минуты для позиции и т.д.) — в соответствии с кодированием в `aprs_ax25_encoder.c`.

### Поведение приложения

- [ ] **Загрузка** при старте FAP (или при входе в главный экран): прочитать `.txt`, разобрать, заполнить структуру конфигурации; при ошибке парсинга — fallback к дефолтам и опционально сообщение в лог (`TAG "APRS432"`).
- [ ] **Запись** при сохранении из настроек (задача 2): перезаписать тот же файл атомарно по возможности (временный файл + rename).
- [ ] Шаблон в репозитории, например: `config/aprs_config.example.txt` или `assets/aprs_config.example.txt` — путь выбрать и не дублировать логику парсера в двух местах.

## Затронутые файлы (ожидаемо)

- Новый модуль: `aprs_config_storage.c` / `aprs_config_storage.h` (или аналогичные имена в стиле проекта).
- `application.fam` — при необходимости указание ресурсов (если шаблон включается в FAP).
- `README.md` — краткая строка о пути к файлу на устройстве (только если уже принято в проекте документировать; иначе достаточно комментария в коде и example-файла).

## Критерии готовности

- Файл на SD открывается в текстовом редакторе и содержит все актуальные поля.
- Изменение файла на ПК с последующей вставкой SD в Flipper отражается после перезапуска приложения (либо по явной команде «Перезагрузить конфиг», если будет добавлена).
