# Задача 1: Экраны настроек APRS Beacon

**Родительская задача**: [task-aprs-spec-compliance.md](task-aprs-spec-compliance.md)  
**Статус**: Новая  
**Приоритет**: Высокий  
**Зависимости**: нет (базовый слой UI)

---

## Цель

Реализовать полный набор экранов конфигурации параметров передачи APRS, чтобы пользователь мог просматривать и переходить ко всем полям без жёстко зашитых лабораторных значений в интерфейсе.

## Контекст проекта

- Приложение: внешнее FAP для Flipper Zero, точка входа — `aprs_hello_world_clean_app.c`, структура состояния — `APRSHelloWorldCleanApp` в `aprs_hello_world_clean_app.h`.
- Сейчас основной экран — мониторинг RX / лабораторная передача; адрес и path заданы константой `aprs_address` (~строка 48 в `aprs_hello_world_clean_app.c`).
- Навигация: очередь событий `EventTypeInput` / тик таймера — паттерн стандартный для fbt-приложений.

## Требования

### Вход в настройки

- [ ] Открытие экрана (или корневого меню) настроек **тройным нажатием кнопки «Вверх»** подряд в основном view.
- [ ] Визуальная обратная связь при распознавании жеста (краткий текст/иконка или переход без ложных срабатываний).

### Экраны и поля (отдельный экран или чёткая подстраница на каждую группу)

Согласно спецификации APRS из родительской задачи:

- [ ] **Позывной (MYCALL)** — до 6 символов, буквы и цифры; отображение ошибки валидации на экране.
- [ ] **SSID** — 0…15.
- [ ] **Позиция** — широта / долгота (ручной ввод; форматирование под APRS — см. задачу по коду передачи).
- [ ] **Направление (bearing)** и **скорость** — отдельные поля ввода в настройках.
- [ ] **Текст статуса** — до 62 символов (отображение лимита/счётчика).
- [ ] **Маршрут (path)** — как минимум path1 (например `WIDE2` + SSID); при необходимости path2; по умолчанию в логике приложения ориентироваться на `WIDE2-2` согласно родительской задаче.

### UX

- [ ] Ясные заголовки экранов (что именно редактируется).
- [ ] Возврат в главный экран и между пунктами меню без потери несохранённого ввода — по согласованию с задачей 2 (если автосохранение — предупреждение не обязательно).

## Затронутые файлы (ожидаемо)

- `aprs_hello_world_clean_app.h` — состояние UI настроек (текущий пункт меню, флаги, буферы отображения при необходимости).
- `aprs_hello_world_clean_app.c` — обработка `InputEvent`, отрисовка через `canvas_*` / `elements_*`, переходы между экранами.

## Критерии готовности

- Пользователь может пройти по всем перечисленным полям из интерфейса приложения.
- Тройной «Вверх» стабильно открывает настройки.
- Валидация позывного видна пользователю до сохранения (согласовано с задачей 2 при вводе с клавиатуры).
