# Задача 2: Ввод с клавиатуры и сохранение конфигурации APRS

**Родительская задача**: [task-aprs-spec-compliance.md](task-aprs-spec-compliance.md)  
**Статус**: Новая  
**Приоритет**: Высокий  
**Зависимости**: желательно после или параллельно с [task-aprs-01-settings-screens.md](task-aprs-01-settings-screens.md)

---

## Цель

Обеспечить ввод каждой настраиваемой переменной через экранную клавиатуру Flipper (цифры и латиница, готовые виджеты API) и надёжное сохранение конфигурации между перезапусками приложения.

## Контекст проекта

- Рекомендуемые компоненты: `TextInput` / аналоги из `gui/modules` (стандартная практика приложений fbt).
- Персистентность на устройстве: типично `storage` + файл на SD (`/ext/...`) или `saved_structures`/NVRAM — согласовать с задачей 3 (читаемый `.txt` как основной контракт для пользователя).
- Текущая структура `APRSHelloWorldCleanApp` не содержит полей для MYCALL, координат и path — их нужно добавить в модель конфигурации и заполнять из результатов ввода.

## Требования

### Клавиатура и ввод

- [ ] Для **каждой** переменной из задачи 1 — отдельный сценарий ввода с клавиатурой (не один общий буфер без подписи поля).
- [ ] Поддержка **цифр и латинских букв** там, где это требуется спецификацией (позывной, статус, path, числовые поля).
- [ ] **Валидация позывного**: максимум 6 символов, допустимые символы по правилам APRS (буквы/цифры) — отклонение с сообщением.
- [ ] **SSID**: 0…15 — ограничение на уровне ввода или немедленная проверка.
- [ ] **Статус**: до 62 символов — счётчик/обрезка по правилам APRS.

### Сохранение

- [ ] После подтверждения ввода значения попадают в **рабочую структуру конфигурации** в памяти приложения.
- [ ] **Сохранение на диск** при выходе из настроек, по кнопке «Сохранить» или автосохранение — зафиксировать один понятный пользователю сценарий.
- [ ] При старте приложения — **загрузка** сохранённой конфигурации; при отсутствии файла — разумные значения по умолчанию (согласовать с задачей 3 и родительской задачей: path по умолчанию `WIDE2-2`).

## Затронутые файлы (ожидаемо)

- `aprs_hello_world_clean_app.h` — структура конфигурации (call, SSID, координаты, bearing, speed, status text, path).
- `aprs_hello_world_clean_app.c` — создание `TextInput`, колбэки, вызовы load/save.
- Модуль загрузки/записи (новый `.c/.h` при необходимости) — тонкая обёртка над файловым API Flipper.

## Критерии готовности

- Все поля можно ввести с устройства и увидеть после перезапуска приложения (через тот же UI или через читаемый файл из задачи 3).
- Нет «тихой» потери данных при нормальном выходе из настроек.
