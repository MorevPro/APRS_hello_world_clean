## Описание проблемы

Текущее приложение передаёт лабораторный beacon с жёстко закодированными тестовыми параметрами:
- Позывной источника: `MYCALL-1` (вместо реального позывного)
- Адресат: `APRS-0` (стандартный, но без настроек)
- Маршрут: `WIDE2-1` (жёстко задан, без возможности изменения)
- Текст статуса: автоматический лабораторный текст (формируется в [aprs_hello_world_clean_app.c](aprs_hello_world_clean_app.c) и [aprs_ax25_encoder.c](aprs_ax25_encoder.c))
- Позиция: отсутствует 
- Телеметрия: отсутствует

**Текущее состояние кода**:
- [aprs_hello_world_clean_app.c](aprs_hello_world_clean_app.c) строка ~50: жёсткие параметры `aprs_address`
- [aprs_ax25_encoder.c](aprs_ax25_encoder.c) строка ~58: функция `aprs_ax25_make_mystatus()` формирует лабораторный статус

### Текущая структура frame кодирования

### Формат Position Report (для справки)

```
!DDMM.hhN/DDDMM.hhW>DDD/SSS/A=AAAAAA comment
```
- `DDMM.hh` — широта в градусах и минутах
- `DDDMM.hh` — долгота в градусах и минутах
- `DDD` — направление (bearing) 0-359°
- `SSS` — скорость в узлах
- `AAAAAA` — высота в футах

## Примечания

**От разработчика** (2026-04-02):
> Текущая версия хорошо работает как лабораторный стенд для проверки RF передачи. Для реальной эксплуатации необходимо:
> 1. Заменить hardcoded параметры на пользовательские настройки
> 2. Добавить позицию (минимум для приёма на APRS-IS)
> 3. Валидировать beacon перед отправкой
> 4. Протестировать на реальных APRS декодерах

**Глюки и особенности**, на которые обратить внимание:
- Bit stuffing в NRZI кодировании уже реализован в `aprs_ax25_emit_byte_nrzi()`
- CRC вычисляется корректно (CCITT/XModem)
- Preamble и postamble флаги установлены согласно spec (32 + 3 флага)
- Модуляция: GFSK 9.99kbps (настроена через `cc1101_preset_gfsk_9_99kb_async_regs`)
