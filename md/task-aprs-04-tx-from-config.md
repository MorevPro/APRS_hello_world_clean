# Задача 4: Передача APRS из сохранённых настроек (encoder + радио)

**Родительская задача**: [task-aprs-spec-compliance.md](task-aprs-spec-compliance.md)  
**Статус**: Новая  
**Приоритет**: Высокий  
**Зависимости**: [task-aprs-03-config-txt.md](task-aprs-03-config-txt.md) или минимум загруженная в память структура из задачи 2

---

## Цель

Убрать лабораторные константы из цепочки формирования кадра: **источник адреса, path, статуса и позиции** — загруженная конфигурация (в т.ч. из `.txt`); обновить AX.25/APRS слой для пользовательских параметров и при необходимости для **Position Report** наряду или вместо чистого status.

## Контекст проекта

- Сейчас: `static const AprsAx25AddressConfig aprs_address` в `aprs_hello_world_clean_app.c` (~строки 48–56); лабораторный текст в `aprs_ax25_make_mystatus()` в `aprs_ax25_encoder.c` (~строка 58).
- Структуры: `AprsAx25AddressConfig` в `aprs_ax25_encoder.h`; буфер кадра `frame_buffer`, передача через `aprs_radio_start_tx()` и сопутствующая логика в `aprs_hello_world_clean_app.c`.
- Родительская задача требует: стандартный APRS статус до 62 символов, произвольный пользовательский текст, корректное форматирование координат (градусы/минуты/секунды по APRS), bearing/speed, path по умолчанию **WIDE2-2**, валидация размера кадра AX.25 (255 байт) — по возможности в рамках этой же задачи или отдельным коммитом после базовой интеграции конфига.

## Требования

### Замена источника данных

- [ ] Удалить или превратить в дефолт **только** `static const` адресную конфигурацию; в рантайме использовать **заполненную из файла/UI** структуру `AprsAx25AddressConfig` (или расширенный тип).
- [ ] `destination` (`APRS-0`) оставить стандартным, если пользователь не задаёт иное — согласно родительской задаче.
- [ ] Path: значения из конфигурации; дефолт при отсутствии файла — **WIDE2-2** (не жёстко `WIDE2-1` в коде без возможности смены).

### Статус и полезная нагрузка

- [ ] Переписать `aprs_ax25_make_mystatus()` (или заменить набор функций): **убрать** строку вида `Flipper Zero lab beacon...`; подставлять **пользовательский статус** из конфигурации.
- [ ] Реализовать/подключить формирование **Position Report** в формате APRS (см. фрагмент в родительской задаче: `!DDMM.hhN/...`) при наличии валидных координат; согласовать с лимитом размера кадра.

### Интеграция с радио

- [ ] Перед передачей собирать кадр из актуальной структуры конфигурации (после `load` при старте и после правок в настройках).
- [ ] Логирование (расширение по родительской задаче): raw hex и декодированное представление — по мере готовности, не блокирует приёмку базовой интеграции конфига.

## Затронутые файлы

- `aprs_hello_world_clean_app.c` — убрать зависимость передачи от константы `aprs_address`; передать указатель/копию конфига в encoder.
- `aprs_ax25_encoder.h` — при необходимости расширить `AprsAx25AddressConfig` и сигнатуры функций формирования INFO/APRS payload.
- `aprs_ax25_encoder.c` — статус, position, валидация длины payload/frame.

## Критерии готовности

- В эфир уходит кадр с **позывным и path из файла/настроек**, а не из зашитых `MYCALL` / `WIDE2-1`.
- Текст статуса соответствует вводу пользователя (в пределах APRS).
- Лабораторная строка в бинарнике не используется как единственный вариант полезной нагрузки.
