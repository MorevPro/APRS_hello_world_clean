# 📋 Задача: Проверка APRS через Direwolf

**Цель:** Настроить приём и декодирование APRS сообщений от Flipper Zero через аудиовыход рации Baofeng на частоте 432.500 MHz, используя микрофонный вход компьютера и Direwolf для декодирования.

**Оборудование:**
- Рация Baofeng (432.500 MHz, AFSK 1200 бит/с)
- Flipper Zero (передатчик APRS)
- Кабель подключения аудиовыхода рации к микрофонному входу компьютера
- ПО Direwolf (Windows/Linux)

---

## 1️⃣ Предварительная проверка и подготовка

### ✅ Проверка оборудования:
```powershell
# На Windows: проверьте устройства ввода (Settings > Sound)
# Должен быть виден микрофонный вход, получающий аудиосигнал от рации
```

**Параметры Baofeng при приёме APRS:**
- Частота: **432.500 MHz**
- Модуляция: **FM** (приём сигнала AFSK)
- Squelch: **открыт** (чтобы слышать шум, когда нет сигнала)
- Громкость: **средняя** (не в минимум, не в максимум)

## 2️⃣ Установка и настройка Direwolf

### Установка Direwolf на Windows:

**Вариант 1: Через Chocolatey**
```powershell
choco install direwolf
```

**Вариант 2: Прямая загрузка**
- https://github.com/wb2osz/direwolf/releases
- Распакуйте в `C:\direwolf\`

### Создание конфига `direwolf.conf`:

Используйте файл `debug/direwolf.conf` в текущей директории или вынесите отдельно (например, `C:\direwolf\`):

```ini
# ============================================================
# Direwolf конфигурация для приёма APRS на 432.500 MHz
# VHF 1200 бит/с AFSK
# ============================================================

# Аудиоустройство (0 = встроенная звуковая карта)
# На Windows: обычно это устройство по умолчанию
ADEVICE 0 0

# Частота дискретизации аудио
# 48000 Hz - оптимально для большинства USB звуковых карт
# 44100 Hz - для встроенной звуковой карты
ARATE 48000

# Канал 0 - приём на VHF 1200 бит/с
CHANNEL 0
MODEM 1200

# Режим работы: только приём (RX only)
PTT NONE

# Порты для подключения других приложений
AGWPORT 8000      # AGW протокол
KISSPORT 8001     # KISS протокол

# Повышение чувствительности к слабым сигналам
DWAIT 0
SLEW 20
```

## 4️⃣ Мониторинг уровня сигнала

### Индикатор уровня в Direwolf:

При правильной конфигурации Direwolf покажет:
```
Audio input level 50% (|||||)  ← хороший уровень сигнала
Audio input level 25% (||)     ← слабый сигнал (может не декодироваться)
Audio input level 100% (||||||||||) ← перегруз (клиппирование, белый шум)
```

**Регулировка:**
- Если уровень < 25%: увеличьте громкость на рации
- Если уровень > 95%: уменьшите громкость (избегайте клиппирования)
- **Оптимум: 40-70%**

## 8️⃣ Что означают декодированные данные

```
[12.5]                    ← время приёма (сек)
KVAS43-1                  ← позывной источника
>APRS                     ← адресат
,WIDE2-2                  ← путь повторения (wide)
:!5530.00N/03700.00E      ← координаты (широта/долгота)
-Your Beacon Message      ← комментарий/статус
```

**Расшифровка координат:**
- `5530.00N` = 55°30'00" северной широты
- `03700.00E` = 37°00'00" восточной долготы (Москва)

## 🔴 Диагностика проблем

### ❌ **Direwolf работает, но НЕ РЕАГИРУЕТ на сигналы (Audio level = 0%)**

**✅ Решение (пошагово):**

#### **Шаг 1: Проверить уровень сигнала в Windows**

Убедитесь, что микрофонный вход получает аудиосигнал:
```
Settings > System > Sound > Volume and device preferences > App volume and device preferences
```
- Выберите микрофон, который подключен к рации
- Если нет сигнала, появится `0%`
- Если есть сигнал, будет зелёная полоска (растёт/падает с звуком)

**Если полоска не меняется:**
1. Убедитесь, что **аудиокабель физически подключён** к микрофонному входу
2. Включите громкость на рации (установите на среднее значение)
3. На рации должны быть **слышны шумы** из наушников (принимаемый сигнал)

#### **Шаг 2: Проверить правильность ADEVICE в direwolf.conf**

Если подключено несколько аудиоустройств, ADEVICE 0 0 может быть неправильным:

```powershell
# Посмотрите в выводе Direwolf при запуске:
direwolf -c direwolf.conf -t 1
```

Должна быть строка:
```
Audio input device 0 (default input)    ← это ваше устройство?
```

Если нет, отредактируйте `direwolf.conf`:
```ini
# Попробуйте ADEVICE 1 1 (если это не ваше устройство)
# Или ADEVICE "USB Audio" "USB Audio" (по названию)
```

#### **Шаг 3: Изменить ARATE (частота дискретизации)**

Многие микрофонные входы работают на 44100 Hz, а не 48000. Попробуйте:

```ini
# Измените в direwolf.conf:
ARATE 44100      # вместо 48000
```

Затем перезапустите Direwolf.

#### **Шаг 4: Проверить, что Flipper Zero действительно передаёт**

**Визуально:**
- На Flipper при передаче APRS должен светиться **индикатор TX** (красный)
- Рация Baofeng должна **войти в режим передачи** (светится красный LED на передатчике)

**На компьютере:**
- Включите звук рации (на динамики), чтобы слышать шум от Flipper
- Должны быть характерные **пищащие звуки** (AFSK модуляция 1200 Hz)

#### **Шаг 5: Отладочная команда с явным выводом уровня сигнала**

```powershell
# Запустите Direwolf в текстовом режиме без графики
direwolf -c direwolf.conf -t 0 -q d -v d | Tee-Object -FilePath direwolf_debug.log
```

Должно вывести все устройства и параметры. Поищите строки:
```
Audio input level 0% (|)    ← если это не меняется, нет сигнала
```

#### **Шаг 6: Прямое тестирование микрофонного входа (обход Direwolf)**

Используйте FFmpeg для записи:
```powershell
# Запишите 10 секунд аудио с микрофона
ffmpeg -f dshow -i audio="Микрофон" -t 10 test_mic.wav

# Затем воспроизведите, чтобы убедиться, что там что-то есть
ffplay test_mic.wav
```

Если в `test_mic.wav` только тишина — **проблема в аудиокабеле или выборе устройства**!

## � Отладка приложения APRS на Flipper Zero

**Если Direwolf успешно принимает сигналы от других устройств (iPhone PulseModem, другие рации), но НЕ принимает от Flipper:**

### ✅ **Проверка настроек Flipper APRS приложения:**

#### **1. Частота передачи**
```
Убедитесь, что установлена: 432.500 MHz (точно!)
Или проверяемая рабочая частота в вашей стране:
- EU: 144.800 MHz (2m APRS)
- US: 144.390 MHz (2m APRS)
- VHF international: 432.500 MHz (70cm APRS)
```

#### **2. Модуляция и скорость**
```
MODEM: AFSK (Audio Frequency Shift Keying)
SPEED: 1200 baud (не 9600!)
TONE: 1200 Hz mark / 2200 Hz space
```

#### **3. Параметры позывного**
```
Callsign: Ваш позывной (например: KV2TST)
SSID: 0-15 (обычно 0 для основной станции, или 1 для мобильной)
```

#### **4. Координаты и статус**
```
Latitude: в градусах (например, 55.8241 или 55°49'26")
Longitude: в градусах (например, 37.5807 или 037°34'50")
Altitude: в метрах (может быть 0 если неизвестна)
Comment: Ваше сообщение (например, "Flipper APRS Test")
```

#### **5. Код модуляции SubGHz**
Flipper использует встроенный передатчик SubGHz. Убедитесь, что:
- **TX (передача) АКТИВНА** - должна быть кнопка/переключатель на экране
- **Мощность передачи**: MAX или HIGH (не LOW)
- **Антенна**: встроенная или внешняя (подойдёт любая)

---

### 🔍 **Как проверить, что Flipper действительно передаёт:**

#### **Метод 1: Визуальная проверка**
```
1. На Flipper при нажатии "Transmit" должны светиться подсказки:
   - Индикатор TX (красный LED или иконка)
   - Счётчик время передачи (обратный отсчёт)

2. На рации Baofeng (если она настроена на 432.500):
   - Должен светиться красный LED передатчика
   - Должны быть слышны характерные пищащие звуки (AFSK)
   - Уровень сигнала на Direwolf должен подняться выше 0%
```

#### **Метод 2: Aудиовизуальная проверка**
```powershell
# Запустите Direwolf и одновременно передавайте с Flipper:
direwolf -c direwolf.conf -t 1

# Должны появиться строки:
# Audio input level XX% (|||||)  ← уровень временно повысится
# [12.5] ...APRS message...       ← декодированное сообщение
```

#### **Метод 3: Запись сигнала для анализа**
```powershell
# Откройте второе окно PowerShell и запишите сигнал:
ffmpeg -f dshow -i audio="Микрофон" -t 20 flipper_signal.wav

# Затем передавайте с Flipper (в течение этих 20 секунд)

# После окончания проверьте размер файла:
Get-Item flipper_signal.wav | Select-Object Length

# Если файл < 100 KB, это скорее всего тишина (нет сигнала)
```

### ❌ **Типичные проблемы с Flipper приложением:**

| Проблема | Решение |
|----------|---------|
| **Flipper не передаёт вообще (TX LED не светит)** | 1. Проверьте, нажали ли кнопку "Transmit" <br> 2. Убедитесь, что режим "TX mode" включен <br> 3. Перезагрузите приложение <br> 4. Проверьте батарею Flipper (может быть разряжена) |
| **Передаёт, но на другой частоте (не 432.500)** | 1. Откройте приложение APRS <br> 2. Перейдите в Settings/Параметры <br> 3. Найдите "Frequency" и установите 432.500 MHz <br> 4. Сохраните и перезагрузитесь |
| **Передаёт, но неправильная модуляция** | 1. Проверьте в приложении MODEM = AFSK <br> 2. Проверьте SPEED = 1200 baud <br> 3. Скорость 9600 работать не будет для APRS! |
| **Сигнал передаётся, но не декодируется** | 1. Проверьте координаты (должны быть в правильном формате) <br> 2. Убедитесь, что статус сообщения корректный <br> 3. Попробуйте изменить мощность передачи (MAX) |
| **Direwolf не видит сигнал от Flipper, но видит от iPhone** | Это означает, что либо Flipper не передаёт, либо передаёт на другой модуляции. Проверьте все параметры выше. |

```

- Callsign: KV2TST
- Frequency: 432.500 MHz
- Modulation: AFSK 1200 baud
- Output: Audio to Baofeng TX

Flipper APRS:
- Должна быть точно такая же конфигурация!
- Если отличается - приложение не будет работать
```
